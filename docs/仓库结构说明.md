# 仓库结构说明

本文档说明 `dev_jaz-may` 分支中已经纳入 Git 版本管理的目录和文件。

## 根目录

| 文件 | 作用 |
| --- | --- |
| `.gitignore` | 配置 Git 忽略规则，避免上传原始数据、中间产物和本地预测结果。 |
| `README.md` | 项目简介、目录说明和 Conda 环境使用方法。 |
| `environment.yml` | 可移植的 Conda 环境配置文件。 |

## `data/`

用于保存本地数据。仓库只提交占位文件，不上传原始数据或处理中间表。

| 路径 | 作用 |
| --- | --- |
| `data/raw/.gitkeep` | 保留本地原始数据目录。原始数据应放在此目录下，但不得上传 GitHub。 |
| `data/processed/.gitkeep` | 保留本地处理后数据目录。生成的数据文件不得上传 GitHub。 |

## `docs/`

项目文档目录。

| 文件 | 作用 |
| --- | --- |
| `docs/.gitkeep` | 保留文档目录。 |
| `docs/背景信息汇总.md` | 汇总赛题背景、预测目标、数据表作用、提交格式、评价指标和注意事项。 |
| `docs/参考思路.md` | 记录基础分析、特征工程、建模、验证和后续改进方向。 |
| `docs/模型优化迭代说明.md` | 详细复盘从原始 LSTM 到约 107 分方案，再到 112 分方案的优化过程。 |
| `docs/仓库结构说明.md` | 说明当前分支中的目录和文件。 |

## `notebooks/`

探索分析和模型实验目录。

| 文件 | 作用 |
| --- | --- |
| `notebooks/.gitkeep` | 保留 notebook 目录。 |
| `notebooks/load_visualization.ipynb` | 分支原有的数据加载与可视化 notebook。 |
| `notebooks/model_baseline_and_lstm_improvement.ipynb` | 基础规则、传统模型和轻量 LSTM 改进实验 notebook。 |
| `notebooks/high_score_periodic_holiday_ensemble.ipynb` | 周期规则、节假日处理和融合预测实验 notebook。 |

## `src/`

可复用的模型和提交文件生成代码。

| 文件 | 作用 |
| --- | --- |
| `src/.gitkeep` | 保留源代码目录。 |
| `src/high_score_periodic.py` | 日粒度聚合、周期规则、大额交易拆分、节假日调整和提交写出工具。 |
| `src/high_score_calendar.py` | 日历特征、Ridge 回归和最终分目标融合方案。 |
| `src/generate_final_submission.py` | 生成本地最终提交文件 `outputs/final/tc_comp_predict_table.csv`。 |

## `outputs/`

本地预测结果目录。除分支历史中已经存在的文件外，不应继续上传新的预测 CSV。

| 文件 | 作用 |
| --- | --- |
| `outputs/.gitkeep` | 保留输出目录。 |
| `outputs/submit_lstm.csv` | 分支原有的 LSTM 预测结果文件。 |

运行以下命令可在本地重新生成最终候选提交文件：

```powershell
python -m src.generate_final_submission
```

生成的 `outputs/final/tc_comp_predict_table.csv` 会被 `.gitignore` 忽略，不会上传 GitHub。
